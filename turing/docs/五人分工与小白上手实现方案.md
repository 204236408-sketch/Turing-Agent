# 五人分工与小白上手实现方案

## 第 1 人：后端接口与项目总装负责人

目标：保证项目能启动、接口能通、前后端能联调。

每天做什么：

第 1 天：安装依赖，启动 FastAPI，确认 `/api/health` 返回 `ok:true`。检查 `main.py` 是否注册所有 router。  
第 2 天：完成登录注册、首页、知识图谱、mastery 接口。用浏览器打开 `/docs` 手动点接口测试。  
第 3 天：完成 `/api/qa/chat` 和 conversation 接口。测试问“LRU 和 FIFO 有什么区别”。  
第 4 天：完成 `/api/questions/generate`、`/api/answers/check`、`/api/questions/mastery`。  
第 5 天：合并 OCR、论坛、报告接口，统一处理异常。  
第 6 天：全流程演示，修复跨域、404、500、接口字段不一致问题。

小白操作步骤：

1. 打开终端进入项目。
2. 执行 `python -m pip install -r backend\requirements.txt`。
3. 执行 `cd backend`。
4. 执行 `python -m uvicorn main:app --reload --host 127.0.0.1 --port 8000`。
5. 打开 `http://127.0.0.1:8000/docs`。
6. 每写完一个接口，就在 `/docs` 点开 Try it out 测试。

验收标准：`/api/health`、登录、出题、批改、错因确认、报告生成全部返回 `ok:true`。

## 第 2 人：数据库与数据负责人

目标：保证表结构、演示数据和掌握状态计算稳定。

每天做什么：

第 1 天：看 `database/schema.sql`，理解 user、knowledge_point、question、answer_record、knowledge_mastery。  
第 2 天：整理 408 四科知识点，补充到 `backend/services/seed_service.py` 或 `backend/data/*.json`。  
第 3 天：准备 10 道演示题，每道题必须有题干、选项、答案、解析、提示。  
第 4 天：重点测试 `knowledge_mastery` 的状态变化：未学、掌握、不熟、不会、薄弱点。  
第 5 天：补论坛帖、错题、长期记忆、视频资源。  
第 6 天：导出最终 SQL，写清楚演示账号。

小白操作步骤：

1. 先运行后端，让系统自动生成 SQLite 数据库。
2. 用 `/api/questions/generate` 生成题。
3. 用 `/api/answers/check` 故意答错。
4. 用 `/api/mistakes/cause-confirm` 确认错因。
5. 用 `/api/mastery/list` 看掌握状态是否变化。

验收标准：演示账号进入系统后首页不空白，知识点、题目、论坛、报告都有数据。

## 第 3 人：RAG、记忆与 Agent 负责人

目标：让系统体现“Agent 会规划、会检索、会记忆”。

每天做什么：

第 1 天：理解 `agents/qa_agent.py` 的四步流程。  
第 2 天：补充 `services/rag_service.py` 的检索规则，让关键词命中更准。  
第 3 天：维护 prompts 目录，把问答、出题、批改、错题、报告 prompt 分开。  
第 4 天：接入硅基流动密钥，修改 `llm_service.py` 的 fallback 文案。  
第 5 天：整理 `user_memory` 的写入规则，保证错因确认后能被问答读到。  
第 6 天：答辩时讲清楚 Agent 流程图。

小白操作步骤：

1. 在 `.env` 填 `SILICONFLOW_API_KEY`。
2. 打开 `/api/health`，确认 `llm.enabled=true`。
3. 调 `/api/qa/chat`，看返回里的 `agent_steps`。
4. 先制造错题记忆，再问同知识点问题，看回答是否带个人提醒。

验收标准：问答返回包含知识点定位、检索结果、长期记忆、后续动作。

## 第 4 人：前端页面与交互负责人

目标：保持 version-b 页面风格统一，并把接口接入做好。

每天做什么：

第 1 天：确认 `version-b.html` 能打开，页面不白屏。  
第 2 天：检查 `app.js` 中 `apiRequest`，确保失败时弹红色提示并降级。  
第 3 天：联调问答页 `/api/qa/chat`。  
第 4 天：联调智能出题、提交答案、错因确认。  
第 5 天：联调论坛 AI 和学习报告。  
第 6 天：检查不同屏幕宽度，保证按钮文字不挤压、不重叠。

小白操作步骤：

1. 启动后端。
2. 打开 `http://127.0.0.1:8000/version-b.html`。
3. 点“智能出题”，生成题目。
4. 选一个答案并提交。
5. 如果接口失败，看右下角报错；如果接口成功，看页面内容是否更新。

验收标准：后端关闭时前端仍能用 Mock 演示；后端开启时能写入真实数据。

## 第 5 人：OCR、论坛、报告与测试负责人

目标：保证外围模块完整，演示路线顺滑。

每天做什么：

第 1 天：测试 OCR 上传 Mock 接口。  
第 2 天：测试 OCR 分析写入错题。  
第 3 天：测试论坛发帖、评论、点赞、AI 小助手。  
第 4 天：测试学习报告生成。  
第 5 天：写 `tests/` 下的接口测试。  
第 6 天：按演示说明走完整流程，记录问题并反馈给对应负责人。

小白操作步骤：

1. 打开 `/docs`。
2. 找 `/api/ocr/upload` 上传任意图片。
3. 找 `/api/forum/posts` 创建帖子。
4. 找 `/api/forum/posts/{post_id}/ai-answer` 生成 AI 回答。
5. 找 `/api/reports/generate` 生成报告。

验收标准：OCR、论坛、报告都能返回 `ok:true`，且不影响主流程。
