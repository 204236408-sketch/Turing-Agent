# P5 测试与验收工作手册

> 角色定位：自动化测试、接口验收、页面验收、部署验收、最终清单。  
> 你不是最后一天才出现的人。你每天都要验证当天最小闭环，防止问题堆到最后。

## 1. 你负责的大模块

| 模块 | 负责内容 | 关键文件 |
|---|---|---|
| 冒烟测试 | 服务能启动、基础接口能访问 | `tests/test_smoke.py` |
| 认证测试 | 注册、登录、token | `tests/test_auth.py` |
| 首页测试 | 今日计划、推荐、统计、记忆 | `tests/test_home_personalization.py` |
| 出题测试 | 生成题目、智能推荐、视频 | `tests/test_question.py`、`tests/test_video.py` |
| 答题测试 | 批改、错因、掌握状态 | `tests/test_answer.py`、`tests/test_mastery.py` |
| 错题测试 | 题本、详情、相似错题 | 新增 `tests/test_mistake.py` |
| 论坛测试 | 搜索、分页、点赞幂等、排行榜 | `tests/test_forum.py` |
| OCR 测试 | 上传、分析、权重、重复 | `tests/test_ocr.py` |
| 报告测试 | 概览、计划、导出 | `tests/test_report.py` |
| 部署验收 | 云端健康检查、MySQL、全链路 | `docs/验收清单.md` |

## 2. 先后顺序和交接关系

```text
P2 给你测试数据
P3 给你 Agent fallback 和 mock 条件
P1 给你接口地址和响应结构
P4 给你页面入口
P5 输出测试结果和验收清单
```

你每天要向其他人反馈：

- 哪个接口失败。
- 是参数问题、数据问题、逻辑问题还是页面问题。
- 阻塞了哪条流水线。
- 是否影响当天交付。

## 3. 测试原则

### 3.1 每天都测最小闭环

不要等第 5 天。

每天至少做到：

- 3 个自动化测试。
- 1 条手工主流程。
- 1 条异常流程。

### 3.2 测试不依赖外部服务

LLM、ChromaDB、爬虫都可能不可用。

测试必须验证：

```text
外部服务不可用时，接口仍返回结构化 fallback。
```

### 3.3 每个接口测三类情况

- 正常参数。
- 错误参数。
- 空数据或降级情况。

## 4. 五天工作安排

### 第 1 天：测试基线和接口契约验收

#### 要做什么

1. 运行现有测试。
2. 记录失败用例。
3. 新增 MySQL 配置检查测试。
4. 检查接口文档是否覆盖：
   - 登录。
   - 首页。
   - 出题。
   - 答题。
   - 错题。
   - 视频。
   - 论坛。
   - 报告。
5. 建立 `docs/验收清单.md` 初版。

#### 重点测试

```text
后端能启动。
/api/health 正常。
演示账号能登录。
未登录访问受保护接口返回 401。
```

#### 交付给谁

- 交给 P1：接口缺失或错误码不统一的问题。
- 交给 P2：缺数据导致无法测试的问题。
- 交给 P3：Agent 返回字段不固定的问题。
- 交给 P4：页面仍显示假数据的问题。

#### 完成标准

- 有测试基线报告。
- 有验收清单初版。
- 每条流水线至少有一个待测入口。

### 第 2 天：首页、视频、收藏测试

#### 要做什么

1. 写或补 `tests/test_video.py`。
2. 写首页推荐测试。
3. 写题目收藏测试。
4. 手工打开首页和出题页视频抽屉。

#### 视频测试场景

```text
有 3 条视频：按 quality_score 返回 top 3。
只有 1 条视频：返回已有视频，crawl_triggered=true。
0 条视频：fallback_used=true，不报 500。
非法 question_id：返回 404。
```

#### 首页测试场景

```text
新用户：返回基线诊断计划。
有错题用户：今日计划优先薄弱点。
统计卡片字段完整。
记忆为空时返回空状态。
```

#### 收藏测试场景

```text
第一次收藏：collected=true。
第二次点击：collected=false 或取消收藏成功。
重复调用不产生多条记录。
```

#### 完成标准

- 视频、首页、收藏都有自动化测试。
- P4 页面能展示视频空状态。
- P1 接口错误码清晰。

### 第 3 天：答题闭环、错题本、掌握状态测试

#### 要做什么

1. 测试生成题目。
2. 测试提交答案。
3. 测试答错后的推荐错因。
4. 测试确认错因。
5. 测试错题本筛选。
6. 测试相似错题。
7. 测试掌握状态标记触发。

#### 答题闭环验收流程

```text
登录
→ 生成题目
→ 提交错误答案
→ 接收 recommended_causes
→ 选择错因并确认
→ mistake 表新增记录
→ user_memory 表新增记录
→ knowledge_mastery 更新
→ 错题本能看到该题
```

#### 掌握状态测试

```text
标记“不熟”：进入不熟题本，weak_score 上升或保持合理。
标记“不会”：进入不会题本，产生推荐动作。
标记“掌握”：状态变掌握，相关错题可标记 resolved。
```

#### 相似错题测试

```text
Chroma 可用：返回语义相似错题。
Chroma 不可用：按同知识点降级返回。
没有相似：返回空数组，不报错。
```

#### 完成标准

- 答题闭环自动化测试通过。
- 手工流程能从出题走到错题本。
- 首页或报告能读取这次答题产生的数据。

### 第 4 天：论坛、报告、OCR、迁移测试

#### 论坛测试

1. 发帖。
2. 搜索。
3. 分类筛选。
4. 分页。
5. 评论。
6. 点赞。
7. 取消点赞。
8. 排行榜。
9. AI 小助手。

重点：

```text
同一用户重复点赞，不允许 like_count 无限增加。
排行榜 daily 和 weekly 都有返回。
搜索不是只查前端当前页数据。
```

#### 报告测试

1. `GET /api/reports/overview`。
2. `POST /api/reports/generate`。
3. 报告训练计划字段完整。
4. 导出接口可用。
5. PDF 不可用时有 print fallback。

#### OCR 测试

1. 图片上传。
2. OCR 不可用时 fallback。
3. 分析后写入错题。
4. 重复 OCR 错题不会无限创建。

#### MySQL 迁移测试

1. 空库建表。
2. 导入种子数据。
3. SQLite 迁移到 MySQL。
4. 对比表数量。

#### 完成标准

- 论坛、报告、OCR、迁移都有测试记录。
- MySQL 空库能导入演示数据。
- 报告页不再依赖假数据。

### 第 5 天：全量回归和云端验收

#### 要做什么

1. 跑全量测试。
2. 云端访问前端。
3. 云端访问 `/api/health`。
4. 云端登录演示账号。
5. 云端跑完整主流程。
6. 输出最终 `docs/验收清单.md`。

#### 云端主流程

```text
登录
→ 首页看到真实数据
→ 开始个性化训练
→ 生成题目
→ 提交答案
→ 确认错因
→ 错题本查看
→ 问答提问
→ 论坛发帖/点赞/排行榜
→ 生成报告
→ 导出报告
```

#### 完成标准

- 本地测试通过。
- 云端测试通过。
- 记录仍未解决的问题和风险。
- 验收清单完整。

## 6. 每日自查清单

- 今天是否新增至少 3 个测试？
- 今天是否跑了至少 1 条主流程？
- 今天是否测了至少 1 个失败/降级场景？
- 问题是否明确归属给 P1/P2/P3/P4？
- 是否更新 `docs/验收清单.md`？

## 7. 你的最终交付物

- 完整测试报告。
- 自动化测试用例。
- 云端验收结果。
- 主流程验收清单。
- 降级测试记录。
- 上线风险清单。
