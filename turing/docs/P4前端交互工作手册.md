# P4 前端交互工作手册

> 角色定位：页面接接口、去 Mock、用户交互、空状态、降级展示、移动端和桌面端验收。  
> 你的上游是 P1 的接口契约，下游是 P5 的页面验收。

## 1. 你负责的大模块

| 模块 | 负责内容 | 关键文件 |
|---|---|---|
| 首页 | 今日计划、推荐、统计、记忆、知识图谱 | `frontend/app.js` |
| 出题页 | 生成题目、答题、批改、错因、视频抽屉 | `frontend/app.js` |
| 错题本 | 不熟/不会题本、详情、相似错题、状态标记 | `frontend/app.js` |
| 问答页 | AI 回答、Agent 步骤、历史会话、追问建议 | `frontend/app.js` |
| 论坛页 | 搜索、分页、发帖、评论、点赞、排行榜、AI 助手 | `frontend/app.js` |
| 报告页 | 真实趋势、画像、权重、计划、导出 | `frontend/app.js` |
| 样式 | 卡片、空状态、响应式、错误提示 | `frontend/styles.css` |

## 2. 先后顺序和交接关系

```text
P2 提供数据样例
P3 提供结构化字段含义
P1 提供接口地址、请求体、返回 JSON
P4 实现页面和交互
P5 验收页面和全链路
```

你不能等所有接口完全完成才开始。做法是：

1. 先按 P1 的接口契约写渲染函数。
2. 接口未完成时用同结构临时数据开发。
3. 接口完成后马上切真实请求。
4. 第 2 天起核心页面不能用固定 Mock 冒充真实数据。

## 3. 前端统一规则

### 3.1 数据来源规则

优先级：

```text
真实接口数据 → 空状态提示 → 错误提示
```

禁止：

```text
接口失败后继续显示固定假数据，让用户以为是真实数据。
```

### 3.2 空状态规则

例如视频为空：

```text
暂无该知识点视频，系统已尝试补充资源。你可以先查看同科目推荐或继续答题。
```

例如错题为空：

```text
当前还没有不会题。完成答题并标记“不会”后会自动出现在这里。
```

### 3.3 错误提示规则

- 用户能处理的错误，用 toast。
- 页面区域数据失败，要显示区域空状态。
- 后端不可用，不允许页面白屏。
- 不能把技术错误堆给用户。

## 4. 五天工作安排

### 第 1 天：找出 Mock 点并准备接口接入

#### 要做什么

1. 打开 `frontend/app.js`。
2. 搜索：
   - `Mock`
   - `toast(`
   - 固定数字，例如 `36`、`75%`、`7 天`
   - 固定知识点，例如 `页面置换算法`
3. 列出需要替换的区域：
   - 首页今日计划。
   - 首页推荐。
   - 首页统计。
   - 首页记忆。
   - 知识图谱状态。
   - 视频抽屉。
   - 错因选项。
   - 错题本详情。
   - 论坛搜索和排行榜。
   - 报告页柱状图、画像、权重、计划。
4. 根据 P1 接口契约写前端数据适配函数。

#### 交付给谁

- 交给 P1：你需要的接口字段清单。
- 交给 P5：页面中哪些区域还没接真实数据。

#### 完成标准

- 输出一份“前端 Mock 替换清单”。
- 所有需要接接口的区域都有函数入口。
- 页面仍能正常打开。

### 第 2 天：首页和视频抽屉接真实接口

#### 要做什么

1. 首页调用 `GET /api/home/overview`。
2. 渲染：
   - `today_plan`
   - `countdown`
   - `recommendations`
   - `stats.cards`
   - `knowledge_graph`
   - `memories`
3. “开始个性化训练”按钮带上今日计划参数跳转到出题页。
4. 视频抽屉调用：
   - `GET /api/questions/{question_id}/videos`
   - 或 `GET /api/videos/recommend?subject=&knowledge_point=`
5. 视频用卡片展示，不再用 toast 连续弹标题。

#### 视频卡片展示字段

- 标题。
- 平台。
- 推荐理由。
- 质量评分。
- 打开链接按钮。

#### 交付给谁

- 交给 P5：首页和视频抽屉验收入口。
- 反馈给 P1：接口字段缺失或命名不一致。

#### 完成标准

- 首页所有核心内容来自后端。
- 视频抽屉可展示 0、1、3 条视频三种状态。
- 无视频时不报错。

### 第 3 天：错题本、批改结果和动态错因

#### 要做什么

1. 出题页提交答案后读取后端返回：
   - `is_correct`
   - `feedback`
   - `answer_record_id`
   - `recommended_causes`
2. 错因选项按 `recommended_causes` 渲染。
3. 确认错因时调用 `POST /api/mistakes/cause-confirm`。
4. 错题本调用 `GET /api/mistakes/notebook?status=不熟`。
5. 不会题本调用 `GET /api/mistakes/notebook?status=不会`。
6. 错题详情调用 `GET /api/mistakes/{id}`。
7. 详情里显示相似错题。

#### 错因展示规则

```text
recommended_causes 前两个标记为“推荐”。
用户仍可多选。
用户可以补充文字说明。
未选择错因不能提交。
```

#### 状态标记展示规则

```text
标记“不熟”：提示“已加入不熟题本，并安排复习”。
标记“不会”：提示“已加入不会题本，并提高推荐优先级”。
标记“掌握”：提示“已标记掌握，将降低推荐权重”。
```

#### 完成标准

- 答错后错因不是固定 8 个。
- 错因确认后页面展示“长期记忆已更新”。
- 错题详情有原题、答案、错因、建议、相似错题。

### 第 4 天：论坛和报告页真实化

#### 论坛要做什么

1. 帖子列表调用：
   - `GET /api/forum/posts?q=&category=&page=&size=`
2. 搜索框不再只过滤当前页面已有帖子。
3. 分类切换重新请求后端。
4. 点赞按钮调用真实接口。
5. 收藏按钮显示收藏状态。
6. 排行榜调用：
   - `GET /api/forum/leaderboard?scope=daily`
   - `GET /api/forum/leaderboard?scope=weekly`
7. AI 小助手展示“结合你的学习情况”段落。

#### 报告要做什么

1. 报告页调用 `GET /api/reports/overview`。
2. 渲染：
   - 总答题数。
   - 正确率。
   - 四科掌握趋势。
   - 学习画像标签。
   - 长期记忆权重。
   - 下一轮训练计划。
3. 训练计划可点击跳转出题页并自动生成题。
4. 导出按钮调用报告导出接口。

#### 完成标准

- 论坛点赞不会前端假加。
- 排行榜不是写死。
- 报告柱状图高度来自后端百分比。
- 训练计划能跳回出题链路。

### 第 5 天：全页面验收和响应式修复

#### 要做什么

1. 按 P5 验收清单逐页走流程。
2. 检查桌面端：
   - 首页。
   - 问答。
   - 出题。
   - 错题本。
   - 论坛。
   - 报告。
3. 检查移动端宽度。
4. 确认接口失败时页面显示空状态。
5. 修复文字溢出、按钮挤压、卡片错位。

#### 完成标准

- 6 个页面都能打开。
- 核心数据都来自后端。
- 后端关掉时不白屏。
- 手机和桌面都没有明显重叠。

## 5. 页面细化规则

### 首页

今日计划点击后必须传：

```json
{
  "subject": "操作系统",
  "knowledge_point": "页面置换算法",
  "difficulty": "中等",
  "question_type": "选择题",
  "count": 3
}
```

### 问答页

Agent 步骤展示：

```text
识别知识点 → 读取记忆 → 检索知识 → 生成回答
```

每步可折叠查看输入摘要和输出摘要。

### 出题页

提交答案后：

```text
正确：展示反馈，允许继续下一题。
错误：展示反馈 + 推荐错因 + 确认错因按钮。
```

### 错题本

列表展示简要信息，详情展示完整信息。不要在列表里堆太多文字。

### 论坛

搜索和分类必须走后端。前端只负责展示当前页数据。

### 报告

所有图表都按后端百分比渲染。无数据时显示“暂无学习记录”，不要显示假柱状图。

## 6. 每日自查清单

- 这个页面的数据是不是来自真实接口？
- 接口失败时有没有空状态？
- 字段为空时页面会不会报错？
- 按钮是否真的调用接口？
- P5 是否知道怎么验收这个页面？

## 7. 你的最终交付物

- 去 Mock 的核心前端页面。
- 首页真实数据渲染。
- 视频抽屉。
- 动态错因。
- 错题本详情。
- 论坛搜索、分页、排行榜。
- 报告真实趋势和导出。
- 全页面空状态和错误状态。
