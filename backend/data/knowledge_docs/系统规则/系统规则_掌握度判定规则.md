## 核心概念

掌握度引擎基于学生的答题记录、错因分析、记忆标记和论坛行为，将一个知识点的掌握状态归为五级：薄弱点 > 不会 > 不熟 > 掌握 > 未学。优先级从高到低，高优先级覆盖低优先级。

## 状态判定优先级（从高到低）

### 等级 0: 未学
触发条件：没有任何行为数据（无答题、无错题、无记忆、无提问、无论坛讨论、无用户标记）。

### 等级 1: 薄弱点
满足以下任一条件：
- 累计答错次数 >= 3
- weak_score（综合弱分）>= 10
- 存在重复同类错因（同一 error_type 出现 >= 2 次）
- 有活跃的弱记忆标记（UserMemory.memory_type == "weak_point"）
- 错题来源类型（input_type）种类 >= 3

### 等级 2: 不会
满足以下任一条件：
- 答题总数 > 0 且正确率 < 50%
- 连续答错次数 >= 2
- 答题数 >= 2 且从未答对
- 存在"概念理解错误"类型的错因

### 等级 3: 不熟
满足以下任一条件：
- 答题总数在 1~2 次之间
- 正确率在 50%~80% 之间（含50%）
- 用户手动标记为"不熟"
- 答错次数为 1 或 2
- 提问次数（qa_count）>= 2

### 等级 4: 掌握
必须同时满足：
- 答题总数 >= 3
- 正确率 >= 80%
- 答错次数 <= 1
- 用户标记不为"不熟"或"不会"
- 无活跃弱记忆
- weak_score <= 2

### 兜底规则
有行为数据但不符合以上任何条件时，保守归为"不熟"。

## weak_score 计算公式

weak_score = max(0,
    wrong_count × 3
    + unfamiliar_count × 2
    + unknown_count × 4
    + ocr_mistake_count × 3
    + rule_confusion_count × 2
    + concept_error_count × 2
    + qa_count
    + forum_count
    - correct_count
    - mastered_count × 2
)

各因子含义：
- wrong_count: 答错总次数（权重3）
- unfamiliar_count: 用户标记"不熟"次数（权重2）
- unknown_count: 用户标记"不会"次数（权重4，最高权重）
- ocr_mistake_count: OCR 输入导致的错题数（权重3）
- rule_confusion_count: 规则混淆类型错因数（权重2）
- concept_error_count: 概念理解错误类型错因数（权重2）
- qa_count: 提问次数（权重1）
- forum_count: 论坛讨论次数（权重1）
- correct_count: 答对次数（负权重，答对减分）
- mastered_count: 用户标记"掌握"次数（负权重，权重2）

## 用户手动标记

用户可通过 `apply_manual_feedback` 手动标记知识点的掌握状态。标记后：
1. 对应错题的 mastery_status 同步更新
2. 最近一条答题记录的 mastery_feedback 同步更新
3. 触发 recalculate_mastery 全量重算

## 同步入口

`synchronize_user_mastery` 可批量同步多个知识点，内部逐个调用 recalculate_mastery。
