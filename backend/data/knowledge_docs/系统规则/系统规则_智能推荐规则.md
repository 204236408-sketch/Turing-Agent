## 核心概念

推荐引擎每次生成 5 种推荐模式，组成用户的"今日计划"。各模式尽量指向不同的知识点，避免重复推荐。

## 五种推荐模式

### 1. 薄弱点强化
- **目标用户**：final_status 为薄弱点/不会/不熟的知识点
- **排序依据**：按 urgency_score 降序
- **去重**：先按"同 mode 4h 内已答过"过滤；剩余再走跨模式 used 软约束
- **典型原因**："薄弱点 · 错 X 次 · 连续错 Y 次，优先生成 填空题/简答题/综合题 专项题"

### 2. 最近错题复练
- **目标用户**：有活跃错题记录（Mistake.status == "active"）
- **选取策略**：取最近一条错题的知识点，若已被同 mode 4h 内用过则顺延下一条
- **题型选择**：由 _to_question_type 根据知识点弱特征决定
- **典型原因**："最近错题来自「知识点名」（错因类型），生成 X 道同类变式题"

### 3. 高频提问专项
- **目标用户**：qa_count > 0 的知识点
- **排序依据**：按 qa_count 降序
- **去重**：先按 mode 4h 内去重，再走跨模式 used 软约束
- **典型原因**："你已围绕该知识点提问 X 次，适合把问懂了的转化为会做题"

### 4. 已改善知识点复测
- **目标用户**：final_status == "掌握" 且未被同 mode 4h 内用过
- **固定设置**：难度"较难"，题型"选择题"，数量 2 道
- **典型原因**："当前状态已掌握（答对 X/Y），提高难度复测掌握是否稳定"

### 5. 四科随机综合
- **选取策略**：基于上海时区日历日计算种子（tm_yday），每周轮换主推科目，每天轮换题型
- **知识点**：从该科目的高频考点中按种子取一个
- **数量**：固定 3 道
- **无行为数据时**：initial=True，用于新用户的基线诊断

## KP 名称归一化（_normalize_kp）

所有 KP 名称进入推荐引擎前先走 _normalize_kp：去首尾空格、去掉 `(xxx)` 括号后缀（中文「（xxx）」与英文「(xxx)」都处理）。

- **入口**：`services/recommendation_service.py::_normalize_kp` / `_kp_key`
- **作用**：OCR 错题回流可能写作「指令集体系结构(ISA)」「进程与线程(同步)」等带括号别名，seed 库标准名是「指令系统」「进程与线程」。不归一化会导致错题与 mastery 失配。
- **历史数据修复**：部署本修复后调用 `normalize_existing_kp_rows(db)` 一次性扫描 KnowledgeMastery / Mistake / QuestionGenerationSession / AnswerRecord，把脏数据修正。函数幂等可重复执行。

## 去重粒度（重要修复）

4 小时去重从「QuestionGenerationSession 生成过」收窄为「AnswerRecord 已提交答案」，且按 mode 分别聚合：

- 旧实现：用户点开"薄弱点强化"按钮但没答完，下一次"高频提问专项"也会跳过该 KP，4 个非综合项全被过滤，今天优先攻克降级到空状态。
- 新实现：4h 内"答过"的 KP 只对"同 mode"去重，跨模式仍以 used 集合做软约束（保证 5 个推荐项尽量不指向同一 KP，但不强制）。

## today_plan 降级链

choose_today_plan 按信号强度递降选择（不是按模式列表顺序）：

1. **最近错题复练**（最高信号：有错题 = 强学习意图）
2. **高频提问专项**（次高信号：用户主动提问过）
3. **薄弱点强化**（中信号：自动定位的弱项）
4. **四科随机综合**（兜底基线诊断）

当所有非综合项都不可用时，回落到"基线诊断"卡片并设置 `initial_state=True`，前端根据这个标志位显示正向引导文案，区别于真实空状态/接口异常。

## 时区

涉及"今日"语义的种子（tm_yday）、4 小时窗口、last_answer_time 差值计算全部使用 `Asia/Shanghai`（`SHANGHAI_TZ`）锚定，避免 UTC 与上海相差 8 小时带来的跨夜错位。

## urgency_score 紧迫度评分

urgency_score = weak_score × 0.4 + continuous_wrong_count × 2.5 + qa_count × 0.5 + recency × 0.5

- recency = max(0, 5 - 距离上次答题天数)，即 5 天内答题越多越紧迫
- 用于对知识点排序，决定哪个知识点最需要优先练习

## 难度选择规则

_to_difficulty(mastery) 逻辑：
- weak_score > 6 → "简单"（越薄弱越出简单题，降低挫败感）
- weak_score >= 3 → "中等"
- total_answer_count >= 5 → "较难"
- 其余返回 base（默认"中等"）

## 题型选择规则

_to_question_type(mastery, mode) 逻辑（按优先级）：
- mode == "已改善知识点复测" → "选择题"
- continuous_wrong_count >= 2 → "填空题"（适合精确回忆）
- qa_count >= 3 → "简答题"（适合深度检验）
- weak_score >= 8 → "综合题"
- 其他 → "选择题"

## 题目数量规则

_to_count(mastery, mode) 逻辑：
- mode == "已改善知识点复测" → 2 道
- mode == "四科随机综合" → 3 道
- weak_score > 6 → 5 道（越薄弱练越多）
- weak_score >= 3 → 3 道
- 其他 → 2 道

## Fallback 机制

- 当某模式无可用数据时：available=False, initial=True，展示引导文案
- resolve_smart_recommendation：如果请求的 mode 不可用，回退到"四科随机综合"
- choose_today_plan：优先取第一个可用的非综合推荐，全部不可用时用"四科随机综合"做基线诊断
