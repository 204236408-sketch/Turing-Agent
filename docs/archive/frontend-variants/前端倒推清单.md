# 研途智学 Day 1 前端倒推清单

## 页面范围

1. 登录 / 注册
2. 首页：完整 408 全局知识图谱、高频重点、薄弱点、推荐训练、错题和问答概览。
   个人掌握状态作为可选图层叠加，不改变知识图谱的完整结构。
3. 知识问答：当前会话、连续追问、从问答生成相关题目
4. 智能出题：筛选、推荐原因、题目、提示、视频、答案、掌握反馈
5. OCR：上传、识别、人工校对、错题分析、确认入库
6. 错题本：历史相似错题、同类错误次数、改善状态
7. 学习报告：正确率、长期薄弱点、反复错误、改善点、下一轮计划
8. 学习画像：长期记忆、知识点权重、学习偏好

## Day 1 必须统一的交互状态

- 首次加载、加载中、成功、空数据、接口失败、Mock 降级。
- 登录失效和 Token 刷新。
- Agent 生成中、流式输出、用户取消、重新生成。
- OCR 识别中、低置信度、人工校对、上传失败。
- 题目生成失败、提示逐层解锁、视频资源为空。
- 记忆更新前是否提示用户，以及用户能否纠正错误记忆。
- 答错后由用户选择真实错因；Agent 推断与用户确认必须分别存储。

## 答错后的错因确认结构

建议接口：`POST /api/mistakes/cause-confirm`

```json
{
  "answer_record_id": 1024,
  "knowledge_point_id": 306,
  "error_types": ["计算错误", "规则混淆"],
  "user_note": "忘记在页面命中时更新最近访问顺序",
  "agent_suggested_types": ["计算错误"],
  "evidence_source": "user_confirmed"
}
```

用户确认的错因可信度应高于 Agent 自动推断，并用于更新
`user_memory.error_pattern` 和知识点薄弱权重。

## 优先接口

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/home/overview`
- `GET /api/knowledge/graph`
- `POST /api/qa/chat`
- `POST /api/questions/generate`
- `POST /api/answers/check`
- `POST /api/ocr/upload`
- `POST /api/mistakes/analyze`
- `GET /api/mistakes`
- `POST /api/reports/generate`
- `GET /api/memory/profile`

## 优先数据库实体

`user`、`knowledge_point`、`conversation`、`conversation_message`、`qa_record`、
`question`、`answer_record`、`mistake`、`video_resource`、`video_view_record`、
`user_memory`、`report`。

## MVP 建议

- 第 1 天前端全部使用固定 Mock JSON。
- 问答先保留单会话，但数据模型必须带 `conversation_id`。
- 知识图谱先用树形结构，避免第一版引入复杂图布局。
- 视频资源先人工维护公开链接。
- OCR 必须保留人工校对。
- 记忆先实现最近 5 轮上下文和 MySQL `user_memory`，ChromaDB 语义记忆后补。
