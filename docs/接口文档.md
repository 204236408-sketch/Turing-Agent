### 题目质量字段（`question` 表 + 序列化返回）

智能出题后，每道题在 API 返回里都带以下质量画像字段，前端据此展示徽标/触发反馈：

| 字段 | 类型 | 含义 |
|---|---|---|
| `is_verified` | bool | 是否通过验证（种子题 1、LLM 题默认 0；进入参考题池的硬门槛） |
| `quality_score` | int (0-100) | 质量分（种子题 100；LLM 题由答题正确率动态更新） |
| `quality_flag` | string | `normal` / `disputed`（已被反馈有误） / `deprecated`（自动下线） |
| `easy_mistakes` | string | 题目"易错点"，由 prompt 生成；展示在题目卡片下方 |
| `source` | string | `seed` / `llm` / `agent_fallback` |
| `hints` | string[] | 分步提示（与"易错点"区别：hints 偏解题步骤，easy_mistakes 偏易踩坑） |
| `sub_questions` | object[] | 综合题的子问题列表 |

### 智能出题 Graph 节点（`agent_steps` 顺序）

```
analyze_user_state → select_target → retrieve_question_context
→ generate_questions → deduplicate → self_check → refill
→ validate_questions → save_questions
或（不通过时）→ fallback_questions
```

- `retrieve_question_context` 只拉 `is_verified=True` 且 `quality_score ≥ 60` 的题作为参考，阻断 LLM 幻觉循环放大
- `self_check` 用 LLM 对照知识库原文判 `pass / fail / uncertain`，`fail/uncertain` 直接剔除
- 累计 3 次 `wrong_answer` 反馈会自动让题目标记 `is_verified=False` + `quality_flag=deprecated`，下次出题不再被参考
