# 仓库结构说明

本结构来自 73 页项目技术方案，并对当前 Git 仓库进行了落地整理。

## 关键决策

| 决策 | 结果 |
|---|---|
| 前端框架 | 使用原生 HTML/CSS/JavaScript，不采用 Vue/React |
| 前端基准 | `frontend/version-b.html` |
| 后端组织 | FastAPI 路由层、服务层、Agent、Prompt 均位于 `backend/` |
| 业务数据 | MySQL |
| 语义数据 | ChromaDB，持久化到 `backend/vector_store/chroma/` |
| Agent 编排 | LangChain + LangGraph |
| OCR | PaddleOCR + OpenCV/Pillow |
| 部署 | Nginx + Uvicorn/Gunicorn |

## 请求流

```text
Browser
  -> frontend/version-b.html
  -> Fetch/Axios /api/*
  -> backend/routers
  -> backend/services
  -> MySQL / ChromaDB / LangGraph / PaddleOCR
  -> external LLM API
```

## 后端边界

- `routers/`：接收参数、鉴权、调用服务，不编写核心业务。
- `services/`：业务规则、数据库事务和外部能力组合。
- `agents/`：LangGraph 流程图。
- `prompts/`：版本化 Prompt 与结构化输出约束。
- `utils/`：无业务状态的通用工具。
- `data/`：可重复导入的初始化 JSON。
- `scripts/`：初始化、导入和重算命令。
- `vector_store/`：ChromaDB 设计与本地持久化目录。
