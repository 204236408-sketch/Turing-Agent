# P4 前端交互工作手册（综合新版）

> 角色定位：页面接真实接口、去 Mock、交互闭环、空状态、错误状态、响应式验收。  
> 上游：P1 API 契约、P3 结构化字段、P2 数据样例。  
> 下游：P5 页面验收。
>
> 注意：本版尽量保留原 P4 Day 1 工作，不大改；Day 2-5 按综合方案补齐 QA/RAG、出题闭环、视频、论坛、报告、OCR。

---

## 1. P4 整体负责范围

| 页面 | 负责内容 | 文件 |
|---|---|---|
| 首页 | 今日计划、推荐、统计、记忆、知识图谱 | `frontend/app.js`、`frontend/styles.css` |
| 问答页 | 结构化回答、Agent 步骤、历史会话、追问建议 | `frontend/app.js` |
| 出题页 | 自由出题、智能推荐、题目渲染、提示、视频、答题 | `frontend/app.js` |
| 错题本 | 不熟/不会题本、详情、相似错题、掌握标记 | `frontend/app.js` |
| OCR | 上传、识别、人工校正、分析结果、重复标签 | `frontend/app.js` |
| 论坛 | 搜索、分页、发帖、评论、点赞、收藏、排行榜、AI | `frontend/app.js` |
| 报告 | 统计、趋势、画像、权重、计划、导出 | `frontend/app.js` |
| 响应式 | 桌面/移动端布局 | `frontend/styles.css` |

如果实际入口仍是根目录：

```text
app.js
styles.css
version-b.html
```

则必须和 `frontend/` 同步，或让 P1 统一只部署 `frontend/`。

## 3. 前端统一规则

### 3.1 数据展示优先级

```text
真实接口数据
→ 明确空状态
→ 明确错误状态
```

禁止：

```text
接口失败后显示固定假数据，让用户以为是真实数据。
```

### 3.2 空状态示例

视频为空：

```text
暂无该知识点视频，系统已尝试补充资源。你可以先查看同科目推荐或继续训练。
```

错题为空：

```text
当前还没有不会题。答题后标记“不会”的题目会自动出现在这里。
```

报告为空：

```text
暂无足够学习记录。完成一次训练后即可生成学习报告。
```

### 3.3 结构化回答展示规则

QA 不能只把一段 HTML 塞进页面。应拆成：

- 直接回答。
- 分步解释。
- 408 考点。
- 常见错误。
- 推荐动作。
- Agent 步骤。

## Day 2：首页、知识图谱、QA 结构化

### 首页要做

接口：

```text
GET /api/home/overview
```

渲染字段：

```text
today_plan
countdown
recommendations
stats.cards
knowledge_graph
memories
```

今日计划按钮点击：

```js
startPersonalTraining(today_plan)
```

传给出题：

```json
{
  "subject": "操作系统",
  "knowledge_point": "页面置换算法",
  "mode": "薄弱点强化",
  "difficulty": "中等",
  "question_type": "选择题",
  "count": 3
}
```

### 知识图谱要做

接口：

```text
GET /api/knowledge/graph?scope=all
```

状态颜色：

```text
未学 #9aa5b1
掌握 #27a978
不熟 #d9a441
不会 #e17843
薄弱点 #e95f52
```

### 问答页要做

接口：

```text
POST /api/qa/chat
GET /api/conversation/{id}/context
```

结构化展示：

```text
direct_answer
step_by_step
exam_focus
common_mistakes
practice_suggestion
agent_steps
suggested_followups
```

Agent 步骤可折叠：

```text
识别目标
读取历史
检索知识库
读取长期记忆
生成回答
```

### 完成标准

- 首页不再显示固定统计假数据。
- QA 页面展示结构化回答。
- 接口失败显示空状态，不显示假回答。

## Day 3：出题、批改、错题本

### 出题页要做

接口：

```text
GET  /api/knowledge/graph
GET  /api/questions/recommendations
POST /api/questions/generate
POST /api/questions/generate-smart
GET  /api/questions/{id}/adaptive-hints
```

要求：

- 自由出题知识点列表来自后端。
- 智能推荐隐藏不可用项。
- 题目渲染支持选择、填空、简答、综合题。
- 动态提示按后端返回展示。

### 批改结果要做

接口：

```text
POST /api/answers/check
```

展示：

```text
是否正确
得分
反馈摘要
标准答案
关键原因
下一步建议
recommended_causes
```

### 错因确认要做

接口：

```text
POST /api/mistakes/cause-confirm
```

规则：

- `recommended_causes` 前两个显示“推荐”。
- 用户可多选。
- 未选择错因不能提交。
- 提交后显示“长期记忆已更新”。

### 错题本要做

接口：

```text
GET /api/mistakes/notebook?status=不熟
GET /api/mistakes/notebook?status=不会
GET /api/mistakes/{id}
POST /api/questions/{id}/mastery
```

详情展示：

- 原题。
- 用户答案。
- 标准答案。
- 错因。
- 建议。
- 相似错题。

### 完成标准

- 主流程不再出现“本地演示题无法写入后端”。
- 答错后能确认错因。
- 错题本能真实显示不熟/不会。
- 标记掌握后从题本移出。

## Day 4：视频、论坛、报告、OCR

### 视频抽屉

接口：

```text
GET /api/questions/{id}/videos
GET /api/videos/recommend?subject=&knowledge_point=
POST /api/videos/view/{video_id}
```

卡片字段：

- 标题。
- 平台。
- 推荐理由。
- 质量评分。
- 时长。
- 打开按钮。

禁止用 toast 一条条弹视频标题。

### 论坛

接口：

```text
GET  /api/forum/posts?q=&category=&page=&size=
POST /api/forum/posts/{id}/like
POST /api/forum/posts/{id}/unlike
POST /api/forum/posts/{id}/collect
GET  /api/forum/leaderboard?scope=daily
GET  /api/forum/hot
POST /api/forum/posts/{id}/ai-answer
```

要求：

- 搜索走后端。
- 分类走后端。
- 点赞和收藏显示真实状态。
- 排行榜不是写死。
- AI 回答高亮“结合你的学习情况”。

### 报告

接口：

```text
GET /api/reports/overview
POST /api/reports/generate
GET /api/reports/{id}/export?format=pdf
```

展示：

- 四维统计。
- 四科掌握趋势。
- 学习画像标签。
- 长期记忆权重。
- 结构化训练计划。
- 导出按钮。

训练计划点击后自动出题。

### OCR

展示：

- OCR 引擎状态。
- 识别文本。
- 人工校正。
- 重复错题标签。
- weak_score 变化。
- 写入记忆结果。

### 完成标准

- 视频、论坛、报告、OCR 不显示固定假数据。
- 论坛点赞不前端假加。
- 报告图表高度来自后端。
