# P1 后端总装与接口工作手册（综合新版）

> 角色定位：后端总装、API 契约、路由编排、MySQL 切换、云部署。  
> 上游：P2 数据库与数据资源、P3 Agent/RAG/爬虫。  
> 下游：P4 前端交互、P5 测试验收。
>
> 注意：本版基本保留原 P1 Day 1 工作，只在后续天数补齐 PDF 差距审计中要求的 MySQL、Chroma/RAG、LangGraph 接口适配、视频爬虫、论坛、报告导出和云部署任务。

---

## 1. P1 整体负责范围

P1 不负责写复杂算法，也不负责准备题库数据。P1 的核心价值是把 P2 和 P3 的产物包装成稳定接口，让 P4 能接、P5 能测、云端能跑。

| 负责部分 | 具体职责 | 主要文件 |
|---|---|---|
| 路由总装 | 所有 router 注册、接口参数校验、返回格式稳定 | `backend/main.py`、`backend/routers/*.py` |
| API 契约 | 请求体、返回 JSON、错误码、是否需要登录 token | `backend/schemas.py`、`backend/utils/response.py`、`docs/接口文档.md` |
| MySQL 切换 | 支持 `MYSQL_URL`、MySQL Docker、云端连接 | `backend/config.py`、`backend/database.py`、`deploy/docker-compose.yml` |
| 视频接口 | 推荐、列表、爬虫触发、点击统计 | `backend/routers/video_router.py`、`backend/routers/question_router.py` |
| 出题接口 | 出题、智能推荐、提示、视频、收藏、交互记录 | `backend/routers/question_router.py` |
| 答题/错题接口 | 批改返回、错题详情、相似错题、题本筛选 | `backend/routers/answer_router.py`、`backend/routers/mistake_router.py` |
| 问答接口 | 结构化 QA、会话上下文、摘要、追问建议 | `backend/routers/qa_router.py`、`backend/routers/conversation_router.py` |
| 论坛接口 | 搜索分页、点赞收藏幂等、排行榜、打卡 | `backend/routers/forum_router.py` |
| 报告接口 | 概览、历史、结构化计划、导出 | `backend/routers/report_router.py` |
| 部署 | Nginx、systemd、Docker Compose、日志、环境变量 | `deploy/*`、`docs/云部署说明.md` |

## 2. 上下游先后顺序


6. 是否需要登录。

## 3. 统一接口规则

所有接口必须统一返回。

成功：

```json
{
  "ok": true,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "ok": false,
  "error": {
    "code": "QUESTION_NOT_FOUND",
    "message": "题目不存在",
    "detail": {},
    "request_id": "..."
  }
}
```

错误码规则：

| 状态码 | 用途 |
|---|---|
| `200` | 成功 |
| `400` | 业务参数错误 |
| `401` | 未登录、token 过期 |
| `403` | 无权限 |
| `404` | 资源不存在 |
| `422` | Pydantic 字段校验失败 |
| `500` | 未预期异常，不能用来表达普通业务失败 |
