# 技术方案

本项目按 PDF 方案实现前后端分离结构：前端为原生 HTML/CSS/JavaScript 的 version-b 轻量学习工作台，后端为 FastAPI。开发版默认 SQLite 自动建表和种子数据，便于小组成员马上运行；生产部署时将 `MYSQL_URL` 指向 MySQL，并替换 `chroma_service.py` 与 `ocr_service.py` 为真实 ChromaDB、PaddleOCR 实现。

## Agent 闭环

1. 用户登录后进入首页，读取知识图谱、掌握状态和今日推荐。
2. 知识问答读取长期记忆、检索知识点，生成结构化回答。
3. 智能出题根据自由选择或智能推荐生成题目批次。
4. 答题批改写入 `answer_record`，刷新 `knowledge_mastery`。
5. 用户确认错因后写入 `mistake` 和 `user_memory`。
6. 学习报告聚合答题、错题、问答、论坛和长期记忆。

## 稳定性设计

- 接口统一响应，前端统一捕获错误并降级到本地 Mock。
- 后端异常统一带 `request_id`，便于排查。
- 默认种子数据保证演示不空白。
- 业务逻辑放在 services/agents，routers 只负责接口收发。
