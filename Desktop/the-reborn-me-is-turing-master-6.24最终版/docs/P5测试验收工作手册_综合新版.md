## 0. P5 无痛执行区

这一节是 P5 每天开始前先看的执行卡。P5 不是最后一天才测试的人，而是每天判断“这件事能不能被别人继续使用”的验收负责人。

### 0.1 今天开始前先确认

| 要确认的事 | 找谁拿 | 为什么 |
|---|---|---|
| 今天主流程 | 组长 | 明确今天验收哪条链路，不被零散功能带偏 |
| 数据样例和导入结果 | P2 | 判断失败是数据问题还是接口问题 |
| Agent 正常和降级样例 | P3 | 写正常/无 LLM/无 Chroma/爬虫失败测试 |
| 接口文档 | P1 | 按契约测，不按口头说法测 |
| 页面操作路径 | P4 | 手工验收能复现，不只看截图 |

### 0.2 P5 每天固定执行顺序

1. 上午跑一次现有测试，记录基线。
2. 根据今天主流程，先写最小自动化测试。
3. 下午 16:00 跟着 P1/P4 跑一条页面或接口主流程。
4. 晚上跑自动化测试，手工点页面，记录失败项。
5. 每个失败项必须写责任人、原因、是否阻塞明天、建议处理方式。
6. 不接受“应该好了”；只接受可复现命令、截图、日志或页面路径。

### 0.4 P5 完成标准

| 层级 | 完成标准 |
|---|---|
| 自动化 | 至少覆盖当天主流程的正常、错误参数、空/降级场景 |
| 手工 | 能按页面路径复现，不用开发者口头辅助 |
| 归属 | 每个失败项能归到 P1/P2/P3/P4 或外部依赖 |
| 风险 | 写清是否阻塞明天、是否影响演示、是否可降级 |
| 文档 | `docs/验收清单.md` 当天更新，别人第二天能直接看 |

不算完成：

- 只说“测试失败”，没有复现方式。
- 只跑接口，不跑当天页面主流程。
- 发现问题但不归属责任人。
- 忽略降级场景，等 Day 5 才发现外部依赖不可用。

### 0.5 新 P5 接手时怎么做

1. 先打开昨天 `docs/验收清单.md`，看未关闭失败项。
2. 跑 `python -m pytest`，确认当前基线。
3. 找组长确认今天必须验收的主流程。
4. 找 P1/P4 要接口和页面路径，找 P2/P3 要样例。
5. 先补能保护主流程的测试，再补边角测试。

### 0.6 P5 遇到阻塞怎么处理

P5 最常见阻塞是无法复现、缺测试数据、接口文档和实际返回不一致、页面没有真实入口。

```text
阻塞点：
无法验收的主流程：
缺谁的交付物：
复现命令或页面路径：
是否阻塞明天：
建议降级方式：
```

如果某个失败项 30 分钟内没有责任人，P5 必须把它升级给组长，不能写成“待确认”过夜。

| 测试范围 | 内容 | 文件 |

## 2. 测试原则

### 2.1 每天都测最小闭环

不要等 Day 5。

每天至少：

- 3 个自动化测试。
- 1 条手工主流程。
- 1 条异常/降级流程。

### 2.2 外部依赖必须测降级

必须测试：

```text
无 LLM
无 ChromaDB
视频爬虫失败
OCR 不可用
MySQL 重启
后端异常
```

### 2.3 每个接口三类测试

```text
正常数据
错误参数
空数据/降级
```

## Day 3：答题闭环和错题本测试

新增测试：

```text
tests/test_question_graph.py
tests/test_answer_graph.py
tests/test_cause_recommendation.py
tests/test_mastery_rules.py
tests/test_mistake_notebook.py
tests/test_mistake_similar.py
tests/test_adaptive_hints.py
tests/test_favorite_question.py
```

### 自动化主流程

```text
登录
→ 生成题目
→ 提交错误答案
→ 返回 recommended_causes
→ 确认错因
→ mistake 新增
→ user_memory 新增
→ knowledge_mastery 更新
→ 错题本可见
```

### 掌握状态测试

```text
点击“不熟” -> 进入不熟题本
点击“不会” -> 进入不会题本
点击“掌握” -> 从题本移出
连续答对 -> 状态下降或移出
薄弱点 -> 只作为知识点级标签，不是题本
```

### 相似错题测试

```text
Chroma 可用 -> 返回相似错题
Chroma 不可用 -> 同知识点 fallback
无相似 -> []
```

### 完成标准

- 答题闭环自动化通过。
- 手工能走到错题本。
- 推荐错因不是固定 8 个。

### 视频测试

```text
3 条视频 -> 按 quality_score 返回 top 3
1 条视频 -> 返回已有，crawl_triggered=true
0 条视频 -> fallback_used=true，不 500
爬虫失败 -> 返回已有种子或空状态
```

### 论坛测试

```text
搜索 -> 后端搜索，不只过滤当前页
分类 -> 后端筛选
点赞 -> 同一用户重复点赞不增加
取消点赞 -> like_count 减少但不小于 0
排行榜 -> daily/weekly 都有 items
论坛 AI -> 返回 personalized_section
```

### 报告测试

```text
overview -> stats/subject_trends/profile/memory_weights/next_plan
plan -> 每项有 subject/knowledge_point/count/difficulty
export -> PDF 或 print fallback
```

### OCR 测试

```text
上传图片 -> 保存路径
OCR 不可用 -> fallback
分析 -> 写 mistake/user_memory
重复题 -> 不无限新增
```

### 完成标准

- 外部爬虫失败不影响视频接口。
- 论坛点赞幂等。
- 报告计划可用于出题。
- OCR 可降级。

```text
打开 http://服务器IP/version-b.html

→ 生成题目
```

```text
关 LLM Key -> 主流程可跑
停 ChromaDB -> QA fallback
模拟爬虫失败 -> 视频 fallback
OCR 不可用 -> 手动校正可用
重启 MySQL -> 数据不丢
```

```text
docs/验收清单.md
docs/上线风险清单.md
```

上线风险清单格式：

```md
| 风险 | 影响 | 归属人 | 是否阻塞上线 | 降级策略 |
|---|---|---|---|---|
```

- 本地测试结果完整。
- 云端验收结果完整。
