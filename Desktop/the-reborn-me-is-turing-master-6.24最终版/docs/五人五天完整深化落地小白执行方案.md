## 0. 总体原则

1. 当前项目已经能跑，五天内不要重写前端和后端框架，只做深化改造。
2. 所有新增能力必须保留 fallback：无 LLM、无 ChromaDB、视频爬虫失败、OCR 不可用时，系统仍能完成主流程。
3. 前端只保留必要空状态，不再用固定假数据冒充真实数据。
4. SQLite 只作为本地开发兜底，最终云端必须使用 MySQL。
5. LangChain/LangGraph 优先接入核心学习链路：知识问答、智能出题、答题批改、错题分析。
6. “不熟/不会题本”按题目级状态管理，“薄弱点”按知识点级状态管理。


## 8. 掌握状态与题本规则

### 8.1 知识点级状态

`knowledge_mastery.final_status` 只用于知识图谱、首页、推荐、报告。

状态：

```text
未学
掌握
不熟
不会
薄弱点
```

推荐优先级：

```text
薄弱点 > 不会 > 不熟 > 掌握 > 未学
```

`weak_score`：

```text
wrong_count * 3
+ unfamiliar_count * 2
+ unknown_count * 4
+ ocr_mistake_count * 3
+ concept_error_count * 2
+ rule_confusion_count * 2
+ qa_count * 1
+ forum_count * 1
- correct_count * 1
- mastered_count * 2
```

最终判断：

```text
无任何学习行为 -> 未学
weak_score >= 10 或 repeated_error_type >= 2 -> 薄弱点
unknown_count >= 1 或 correct_rate < 50% 或 continuous_wrong_count >= 2 -> 不会
unfamiliar_count >= 1 或 50% <= correct_rate < 80% 或 wrong_count in [1,2] -> 不熟
total_answer_count >= 3 且 correct_rate >= 80% 且 weak_score <= 2 -> 掌握
其他 -> 不熟
```

### 8.2 题目级题本状态

`mistake.mastery_status` 或最新 `answer_record.mastery_feedback` 用于不熟/不会题本。

规则：

```text
点击“不熟” -> 进入不熟题本
点击“不会” -> 进入不会题本
点击“掌握” -> 从不熟/不会题本移出，mistake.status = resolved
同一道题复练答对 2 次 -> 不会降为不熟
同一道题连续答对 3 次 -> 移出题本
```

说明：

- “薄弱点”不是题本，是知识点级风险标签。
- 一个知识点可以是薄弱点，但其中某一道题可以已经掌握。

## 9. 视频爬虫规则

### 9.1 质量评分

```text
quality_score =
platform_weight * 0.30
+ relevance_weight * 0.40
+ popularity_weight * 0.20
+ freshness_weight * 0.10
```

`platform_weight`：

```text
B站官方合作 UP 主 = 100
B站普通 UP 主 = 80
中国大学MOOC = 90
慕课网 = 85
YouTube = 70
```

`relevance_weight`：

```text
标题包含目标知识点 = 100
标题包含同义词 = 80
描述包含知识点 = 60
否则 = 30
```

`popularity_weight`：

```text
log2(播放量) / log2(max_play) * 100
无法获取播放量 = 50
```

`freshness_weight`：

```text
max(0, 365 - 发布天数) / 365 * 100
```

### 9.2 限流

```text
每个 knowledge_point 每小时最多爬 1 次
每个 IP 每小时最多手动触发 5 次
数据库已有 >=3 条视频时不实时爬
数据库有 1-2 条时后台补充
数据库 0 条时返回同科目 fallback，并后台爬取
```
