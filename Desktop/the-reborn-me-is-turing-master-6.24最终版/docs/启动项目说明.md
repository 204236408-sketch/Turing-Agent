# 项目启动说明

## 1. 进入项目目录

```powershell
cd C:\Users\Sophia\Desktop\test\turing
```

## 2. 安装后端依赖

第一次启动前执行：

```powershell
python -m pip install -r backend\requirements.txt
```

如果提示 `Requirement already satisfied`，说明依赖已经安装好，可以继续下一步。

## 3. 启动后端

```powershell
cd backend
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

看到类似下面的信息，表示后端已经启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
Application startup complete.
```

## 4. 打开前端页面

浏览器访问：

```text
http://127.0.0.1:8000/version-b.html
```

也可以访问根路径：

```text
http://127.0.0.1:8000/
```

前端页面会自动请求同一个后端的 `/api` 接口，不需要单独启动前端服务。

## 5. 演示账号

```text
账号：demo@turing408.ai
密码：123456
```

## 6. 检查后端是否正常

浏览器访问：

```text
http://127.0.0.1:8000/api/health
```

如果返回内容里有：

```json
{
  "ok": true
}
```

说明后端接口正常。

## 7. 停止项目

在启动后端的终端窗口里按：

```text
Ctrl + C
```

如果后端是在后台启动的，可以先查看 8000 端口对应的进程：

```powershell
Get-NetTCPConnection -LocalPort 8000 | Select-Object OwningProcess
```

再结束对应进程：

```powershell
Stop-Process -Id 进程ID
```

## 8. 常见问题

### 端口 8000 被占用

可以换一个端口启动，例如：

```powershell
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8001
```

但当前前端默认请求 `http://127.0.0.1:8000`，如果换端口，需要同步修改前端里的接口地址。

### 页面能打开，但数据加载失败

先访问：

```text
http://127.0.0.1:8000/api/health
```

如果健康检查打不开，说明后端没有启动成功；如果健康检查正常，再刷新前端页面。

### 大模型相关功能不可用

检查 `backend\.env` 是否配置了：

```text
SILICONFLOW_API_KEY=你的密钥
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V3
```

未配置密钥时，普通页面和本地数据接口仍然可以使用，但依赖大模型的功能可能无法正常返回真实结果。
