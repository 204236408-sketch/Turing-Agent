一、登录 / 注册
================================================================================
  ● 页面:    登录页（应用入口）
  ● 区域:    登录表单 + 创建新账号按钮
  ● 当前写死内容:
    - 输入框预填 demo@turing408.ai / 123456
    - 点击"进入图灵学习空间"直接调用 enterApp() → 渲染 shell 主页，无任何认证
    - "创建新账号"只弹 Toast 提醒，不调用注册接口
    - 底部提示"演示账号已自动填充 · 数据仅保存在本地"
  ● 应接接口:
    - POST /api/auth/login      （获取 token 存入 localStorage）
    - POST /api/auth/register   （注册新账号）
    - GET  /api/auth/me         （校验登录态，自动跳回登录页当 token 失效）
  ● 优先级:  P0（必须 — 没有用户认证，所有用户数据无法隔离）
  ● 负责人:  待定

================================================================================
二、个人账户设置（侧面板）
================================================================================
  ● 页面:    右上角头像 → "个人账户"侧面板
  ● 区域:    昵称修改 + 密码修改
  ● 当前写死内容:
    - 昵称默认"林同学"，saveName() 只更新 DOM (#topUserName, #topAvatar)，不调 API
    - 密码修改只做前端校验（长度 ≥ 8），不调任何 API
    - 刷新页面后所有修改丢失
    - 打开面板时未加载后端真实数据
  ● 应接接口:
    - GET  /api/profile/overview  （打开面板时加载最新个人信息）
    - PUT  /api/profile/update    （保存昵称等修改）
    - （密码修改需后端增加对应接口，或扩展 PUT /api/profile/update）
  ● 优先级:  P1（重要 — 昵称/密码无法持久化，学习画像依赖个人信息）
  ● 负责人:  待定

四、侧边栏记忆芯片（Memory Chip）
================================================================================
  ● 页面:    左侧导航栏底部
  ● 区域:    .memory-chip 元素
  ● 当前写死内容:
    - "🔥 连续学习 7 天"（硬编码）
    - "今天再学习 18 分钟，即可完成本周目标"（硬编码）

    - GET /api/home/overview       （获取今日学习进度）
  ● 优先级:  P3（较低 — 功能展示性，不影响核心操作，但影响用户体验）
  ● 负责人:  待定
